# Silk（丝路）本地AI中转客户端 — 技术选型方案
## 整体定位
Tauri + Rust 高性能后端网关 + Vue3 可视化桌面GUI，基于Bun前端构建，实现：三大LLM协议双向互转、图像/视频接口透传、统一本地网关、全可视化配置、日志持久化、托盘后台驻留。
所有依赖经过对比调研，剔除冗余库、替换老旧组件，兼顾**开发效率、运行性能、跨平台稳定性、打包体积**。

## 一、Rust 后端（src-tauri/Cargo.toml 推荐依赖）
## 1. 运行时 & HTTP网关核心
### tokio
- 作用：异步 IO 运行时，支撑并发请求、SSE 长流、数据库读写、后台网关常驻
- 不可替代：Tauri、axum、reqwest、sqlx 全系生态基于 Tokio，切换 async-std 会产生大量兼容问题

### axum
- 作用：本地HTTP网关服务（监听127.0.0.1），请求路由分发、鉴权中间件、SSE流式输出、路径匹配区分LLM/媒体接口
- 选型理由：原生适配Tokio，Tower中间件生态成熟，反向代理、协议转换开源案例最多；不换Actix-web/Salvo，避免运行时冲突与社区薄弱问题

### hyper-rustls
- 作用：纯Rust实现TLS/HTTPS，无系统OpenSSL依赖，打包后跨Windows/macOS/Linux运行无缺失组件

## 2. 上游请求转发 & SSE流式处理
### reqwest（开启rustls、socks特性）
- 作用：异步HTTP客户端，转发请求至OpenAI/Claude/通义/SD/文生视频；支持图片/视频multipart上传、HTTP/SOCKS5代理、连接池、超时控制
- 优化点：移除独立`tokio-socks`，由reqwest内置代理能力统一实现，减少一个依赖

### sse-reqwest-client
- 作用：解析大模型SSE打字流，自动处理断线重连、Last-Event-ID、分片数据组装
- 替换方案：淘汰老旧、维护停滞的`reqwest-eventsource`，该库标准兼容性差、异常处理简陋

## 3. JSON协议转换（核心业务：三模型互转）
### serde + serde_json + serde_with
- 作用：强类型序列化/反序列化
  1. OpenAI ↔ Claude ↔ Qwen 请求/响应报文双向字段映射转换
  2. 本地配置、日志数据持久化序列化
- 选型理由：编译期类型校验，提前拦截协议转换错误；不换simd-json，后者Windows兼容性差，个人桌面无性能收益

## 4. 本地持久化存储（密钥/规则/调用日志）
### sqlx（仅启用sqlite特性）
- 作用：异步操作SQLite本地数据库，存储服务商密钥、转发规则、全量请求日志，支持分页检索、条件查询
- 优化点：移除冗余`rusqlite`，单库统一同步/异步读写；不更换redb/sled键值库，KV库无SQL检索能力，日志分页、筛选开发成本极高

## 5. 工具辅助库
### thiserror
- 作用：统一自定义错误枚举，协议解析失败、网络超时、密钥错误标准化输出，推送到前端日志面板，Rust生态标准无更好替代

### chrono
- 作用：时间戳生成、日志时间格式化、接口耗时计算

### dirs
- 作用：跨系统获取标准配置目录，自动存放SQLite数据库、日志文件，不用手动处理不同系统路径差异

## 6. Tauri 桌面原生层（不可替换）
### tauri（内置 tray-icon）+ tauri-plugin-fs / tauri-plugin-window
- 作用：桌面程序底层能力
  1. 原生窗口、最小化到托盘、开机自启、后台静默运行
  2. IPC前后端双向通信（前端下发配置、后端实时推送日志）
  3. 本地配置文件导入导出、系统弹窗提示
- 选型结论：放弃egui/iced/Fyne纯原生GUI，后台表单、日志表格开发工作量会翻倍

## 二、前端技术栈（package.json，推荐依赖）
## 1. 构建基础
### Bun + Vite
- Bun：极速 JS 运行时与包管理器，替代 npm/pnpm，依赖安装、热更新、打包速度大幅提升
- Vite：Vue 构建工具，开发热重载毫秒级响应

### Vue3
- 前端界面核心框架，模板语法快速编写大量配置表单、表格

### @tauri-apps/api
- 前端IPC通信SDK，调用Rust后端所有能力：启停网关、增删服务商、读取实时日志、文件操作

## 2. UI组件（核心界面载体）
### naive-ui（最终选定，不替换）
- 作用：后台管理全套成熟组件，完全匹配本软件页面需求
  - Form：服务商密钥、代理、超时、模型映射配置表单
  - Table：分页实时请求日志，支持高亮报错、复制报文、虚拟滚动
  - Collapse：分组折叠LLM转换规则、图像视频透传规则
  - Modal、开关、下拉、提示、表单校验、完整深色/浅色主题
- 对比结论：体积、自定义主题、TS友好度优于Element Plus / Ant Design Vue，Tauri轻量桌面客户端最优选择

### @iconify/vue
- 海量矢量图标，菜单、设置、日志、转发、代理状态图标，替代笨重FontAwesome

## 3. 页面辅助工具
### vue3-json-viewer
- 格式化、折叠、复制JSON请求/响应报文，日志面板必备组件，无轻量化平替

### axios
- 前端内置调试工具，直接请求本地网关接口，拦截器统一处理报错

### tailwindcss + postcss + autoprefixer
- 辅助布局、间距、自适应样式，轻量化搭配NaiveUI，不冲突原生组件样式

## 三、完整技术栈优劣总结 & 选型依据
## 优势
1. **性能拉满**：Rust+Tokio异步处理海量SSE并发，长时间后台挂机无内存泄漏、无卡顿；
2. **体积小巧**：Tauri打包仅5–30MB，空闲内存30–60MB，远优于Electron；
3. **开发平衡**：Rust负责高性能网关与数据存储，Vue+NaiveUI快速开发复杂可视化配置页面；
4. **依赖精简**：剔除rusqlite、tokio-socks、reqwest-eventsource、url等冗余/老旧库，减少编译体积与潜在bug；
5. **隐私安全**：所有密钥、日志本地SQLite存储，不上传第三方服务器；
6. **全平台兼容**：Windows/macOS/Linux一套代码编译，无系统依赖缺失问题。

## 舍弃方案说明
1. 舍弃Go+Wails：并发SSE、协议转换、长期稳定性弱于Rust；
2. 舍弃纯原生Rust GUI(egui/iced)：缺少成熟表格、表单组件，界面开发成本翻倍；
3. 舍弃LiteLLM/One-API：依赖Web网页管理，不符合你“独立桌面客户端”需求；
4. 舍弃Svelte/Solid前端：UI 组件生态成熟度不足，当前阶段开发配置密集型后台工具效率较低。

## 四、项目完整分层功能对应
1. **网关转发层（axum+reqwest+sse-reqwest-client）**
   接收本地客户端请求、区分LLM/媒体接口、上游代理转发、流式响应回传
2. **协议转换层（serde全家桶）**
   OpenAI/Claude/通义千问报文互相转换，统一中间标准格式
3. **持久化层（sqlx+sqlite）**
   保存渠道密钥、路由转发规则、全量调用日志
4. **桌面底层（tauri）**
   系统窗口、托盘驻留、前后端IPC通信、本地文件读写
5. **可视化界面层（Vue3+NaiveUI）**
   服务商管理、路由规则配置、实时日志查看、全局程序设置
6. **前端构建工具（Bun+Vite）**
   极速开发热更新、打包静态资源

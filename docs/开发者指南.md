## 添加新的协议适配器

当前 `ProviderAdapter` trait 定义（`protocol/adapter.rs`）：

```rust
#[async_trait]
pub trait ProviderAdapter: Send + Sync {
    fn provider_type(&self) -> &'static str;

    /// 将原始请求体（JSON 字节）转为上游请求
    /// req_body 是客户端发送的原始 JSON 字节
    async fn transform_request(
        &self,
        req_body: &[u8],
        provider: &Provider,
        selected_api_key: &str,
    ) -> Result<UpstreamRequest, ProtocolError>;

    /// 将上游响应转为客户端期望的 JSON 格式
    async fn transform_response(
        &self,
        resp: &UpstreamResponse,
    ) -> Result<serde_json::Value, ProtocolError>;
}
```

> **注意**：不再使用 `CanonicalRequest` / `CanonicalResponse` 中间格式，也不使用 `canonicalize_request` / `parse_response` 方法名。协议转换直接操作 JSON。

### 1. 创建适配器文件

在 `src-tauri/src/protocol/adapters/` 下创建新文件：

```rust
// src-tauri/src/protocol/adapters/my_protocol.rs

use async_trait::async_trait;
use axum::http::{HeaderMap, HeaderValue};

use crate::models::Provider;
use crate::protocol::adapter::{ProtocolError, ProviderAdapter, UpstreamRequest, UpstreamResponse};

pub struct MyProtocolAdapter;

#[async_trait]
impl ProviderAdapter for MyProtocolAdapter {
    fn provider_type(&self) -> &'static str {
        "my_protocol"
    }

    async fn transform_request(
        &self,
        req_body: &[u8],
        provider: &Provider,
        selected_api_key: &str,
    ) -> Result<UpstreamRequest, ProtocolError> {
        let body: serde_json::Value = serde_json::from_slice(req_body)
            .map_err(|e| ProtocolError::SerializationError(e.to_string()))?;

        let mut headers = HeaderMap::new();
        headers.insert(
            axum::http::header::AUTHORIZATION,
            HeaderValue::from_str(&format!("Bearer {}", selected_api_key))
                .map_err(|e| ProtocolError::InvalidValue {
                    field: "Authorization".to_string(),
                    reason: e.to_string(),
                })?,
        );
        headers.insert(
            axum::http::header::CONTENT_TYPE,
            HeaderValue::from_static("application/json"),
        );

        Ok(UpstreamRequest {
            url: format!("{}/v1/chat/completions", provider.api_base_url),
            method: "POST".to_string(),
            headers,
            body,
        })
    }

    async fn transform_response(
        &self,
        resp: &UpstreamResponse,
    ) -> Result<serde_json::Value, ProtocolError> {
        if resp.status >= 400 {
            let msg = resp.body["error"]["message"]
                .as_str()
                .unwrap_or("unknown error");
            return Err(ProtocolError::UpstreamError {
                status: resp.status,
                message: msg.to_string(),
            });
        }

        // 直接透传或按需转换
        Ok(resp.body.clone())
    }
}
```

## 添加新的中间件

当前 10 个中间件模块（不含 `stream_response.rs`，它是内部工具模块），每个在 `gateway/middleware/xxx.rs` 中导出一个 `run()` 函数：

| 中间件 | 函数签名 | 说明 |
|--------|---------|------|
| `extract` | `initialize(parts) -> RequestContext` + `read_body(ctx, body) -> Result<RequestContext, StageError>` | 提取请求，读取 body |
| `authenticate` | `run(ctx: RequestContext) -> Result<RequestContext, StageError>` | 网关 Key 认证 |
| `rate_limit` | `run(ctx: RequestContext, runtime: &GatewayContext) -> Result<RequestContext, StageError>` | IP 级限流 |
| `resolve_route` | `run(runtime: &GatewayContext, ctx: RequestContext) -> Result<RequestContext, StageError>` | 路由解析 |
| `select_channel` | `run(ctx: RequestContext) -> Result<RequestContext, StageError>` | 选择上游 Key |
| `transform_request` | `run(ctx: RequestContext) -> Result<RequestContext, StageError>` | 请求协议转换 |
| `dispatch_upstream` | `run(runtime: &GatewayContext, ctx: RequestContext) -> Result<RequestContext, StageError>` | 上游转发 |
| `transform_response` | `run(ctx: RequestContext) -> Result<RequestContext, StageError>` | 响应协议转换 |
| `persist_log` | `run(log_sender, ctx: &mut RequestContext) -> Result<(), GatewayError>` | 异步日志写入 |
| `finalize` | `success(ctx: RequestContext) -> Response` + `failure(error: GatewayError) -> Response` | 构建响应 |

> 大多数中间件接收 `RequestContext`（所有权转移）并返回 `Result<RequestContext, StageError>`。
> `persist_log` 接收 `&mut RequestContext`，`finalize` 不返回 Result。

### 1. 创建中间件文件

在 `src-tauri/src/gateway/middleware/` 下创建：

    // 实现逻辑...

    // 成功则返回 ctx
    Ok(ctx)

    // 失败则返回 StageError
    // Err(StageError::new(error_ctx, GatewayError::BadRequest("...".to_string())))
}
```

```

### 2. 创建 Repository

在 `src-tauri/src/persistence/` 下创建，Repo 是一个空结构体 + impl 块：

```rust
// src-tauri/src/persistence/my_repo.rs

use sqlx::SqlitePool;
use crate::models::{MyModel, NewMyModel};

pub struct MyRepo;

impl MyRepo {
    pub async fn create(pool: &SqlitePool, new: &NewMyModel) -> Result<MyModel, sqlx::Error> {
        let id = uuid::Uuid::new_v4().to_string();
        let now = chrono::Utc::now().naive_utc();

        sqlx::query_as::<_, MyModel>(
            r#"
            INSERT INTO my_table (id, name, created_at)
            VALUES ($1, $2, $3)
            RETURNING id, name, created_at
            "#,
        )
        .bind(id)
        .bind(&new.name)
        .bind(now)
        .fetch_one(pool)
        .await
    }

    pub async fn find_by_id(pool: &SqlitePool, id: &str) -> Result<Option<MyModel>, sqlx::Error> {
        sqlx::query_as::<_, MyModel>(
            "SELECT id, name, created_at FROM my_table WHERE id = $1",
        )
        .bind(id)
        .fetch_optional(pool)
        .await
    }
}
```

## 错误处理

### GatewayError（网关层）

所有网关错误使用 `GatewayError` 枚举（`gateway/error.rs`）：

```rust
pub enum GatewayError {
    BadRequest(String),         // 400
    NotFound(String),           // 404
    Unauthorized(String),       // 401
    Transform(String),          // 400 - 协议转换错误
    Upstream(reqwest::Error),   // 502
    UpstreamError { status: u16, body: Value },  // 上游返回 4xx/5xx
    Timeout,                    // 504
    TooManyRequests,            // 429
    Database(sqlx::Error),      // 500
    Internal(String),           // 500
    Serialization(String),      // 500
}
```

### ProtocolError（协议层）

所有协议转换错误使用 `ProtocolError` 枚举（`protocol/adapter.rs`）：

```rust
pub enum ProtocolError {
    UnsupportedFormat(String),
    MissingField(String),
    InvalidValue { field: String, reason: String },
    UpstreamError { status: u16, message: String },
    SerializationError(String),
    UnsupportedContentType(String),
    Transform(String),
}
```

### ServiceError（应用服务层）

所有 application service 返回 `ServiceError`（`error.rs`），commands 层通过 `.map_err(|e| e.to_string())` 转为 Tauri 兼容的 `String`。

## 管道流程详解

`GatewayPipeline` 在 `execute()` 中编排中间件，含三级失败回退：

```
execute(Request)
  ├─ extract::initialize(parts)           → RequestContext
  ├─ extract::read_body(ctx, body)        → 读取 body（2MB 限制）
  ├─ authenticate::run(ctx)               → 网关 Key 认证（仅 /v1/* 路径）
  ├─ rate_limit::run(ctx, runtime)        → IP 级限流
  ├─ resolve_route::run(runtime, ctx)     → 路由匹配
  │   └─ 若路径为 /v1/models，直接构建响应返回（跳过后续）
  └─ run_with_failover(ctx)               → 三级失败回退循环
       ├─ [Level 3] 换渠道：rate_limit → select_channel → transform → dispatch
       ├─ [Level 2] 换 Key：select_channel 跳过已失败 Key
       └─ [Level 1] dispatch_upstream 内部重试
       ├─ transform_response → Ok
       └─ 均失败 → Err
  ├─ persist_log::run(sender, &mut ctx)   → 异步日志写入
  └─ finalize::success(ctx) 或 finalize::failure(error)
```

## 代码规范

### Rust

- 使用 `thiserror` 定义错误枚举
- 使用 `serde` + `serde_json` 处理序列化
- 异步函数使用 `async fn`
- 错误处理使用 `Result<T, E>` 而非 panic

### 命名

- 文件名：`snake_case.rs`
- 结构体/枚举：`PascalCase`
- 函数/变量：`snake_case`
- 常量：`SCREAMING_SNAKE_CASE`

### 文档

- 公开函数添加 `///` 文档注释
- 复杂逻辑添加行内注释
- 使用中文注释（与项目约定一致）
